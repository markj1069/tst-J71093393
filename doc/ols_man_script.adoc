=== Standard Options

[horizontal]

[*--input*=_input_file_ | *-i* _input_file_]:: If no input files are specified, _i.e._, *--input* or *-i*, the input is read from standard in, _STDIN_. Multiple input files are supported. If no input or output files are specified, *{script_name}* copies _STDIN_ to _STDOUT_ much like cat. If no input file or _STDIN_ exists, *{script_name}* exits gracefully with a exit code of 0.

[*--output*=_output_file_ | *-o* _output_file_]:: If no output file is specified, _i.e._, --output or -o, the output is written to _STDOUT_. If no input or output files are specified, *{script_name}* copies _STDIN_ to _STDOUT_ much like cat. Do not use the same file as an input_file and as an output_file, see <<input_output_restriction,warning>>.

[*--help*]:: Print the help message to standard error, _STDERR_, and exit with an exit code of 0.

[*--log*]:: Log significant events to _STDERR_.

[*--quiet*]:: Only print fatal error messages to _STDERR_.

[*--usage*]:: Print the usage message to standard error, _STDERR_, and exit with an exit code of 0.

[*--debug*]:: Turn on the debug switch.

[*--version*]:: Print the version, copyright, and license message to standard error, _STDERR_, and exit with an exit code of 0.

[*--verbose*]:: Turn on the verbose switch.

[*--*]:: File list marker. The the double dash, `--`, on the command line signals the end options. The remaining items arguments,
even if some look like options.

[_input_files_]:: Additional input files.


== Exit Status

Here is a list of the exit codes that *{script_name}* can return, along with an explanation of each code. The exit code is a number between 0 and 255, where 0 indicates successful completion and any non-zero value indicates an error or warning.

[horizontal]

EX = 0:: Successful completion.

EX = 4:: Warning, but the script completed successfully.

EX = 8:: Error, but part of the script completed okay.

EX &ge; 16:: Fatal Error, the script did not complete successfully.

== Notes & Caveats

[#input_output_restriction]
WARNING: Input file and Output File Restriction

Do not use the same file as an input and output in the same command of *{script_name}*. You will
destroy your data. *{script_name}* checks for *--input* and *--output* being equal; however,
you should not do this in the first place.

[.line-through]#{script_name} --input=_file-one_ > _file-one_#
